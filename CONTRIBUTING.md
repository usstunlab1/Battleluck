# Contributing

Keep pull requests focused, build before submission, and document any V Rising server-version constraint.

- Do not commit secrets, server saves, or build outputs.
- Include tests for configuration, command, lifecycle, and safety behavior.
- Require admin authorization for live spawning, moving, despawning, or configuration mutation.
- Track and clean up every event-owned NPC/entity.
- Respect upstream licences and accurately declare BepInEx and VampireCommandFramework dependencies in published packages.
